# LT-100 RS-232C 通訊協議

## 更新日期
2026-03-18

> 本文件整理自 LT-100DX880 使用手冊 V2，僅擷取與 RS-232C 資料傳輸相關的章節。
> 操作面板、重量校正、荷重元安裝等硬體操作內容不在此文件範圍內。

---

## 📋 概述

LT-100 是一款工業級重量顯示器，透過 RS-232C 或 Current Loop 介面將秤重結果傳送至電腦端。本文件說明其傳輸相關的參數設定、資料格式定義、以及 PC 命令模式。

---

## 一、硬體介面規格

### 1.1 AD 轉換特性

| 項目 | 規格 |
|------|------|
| 最大零點可調範圍 | 0mV ~ 20mV |
| 輸入靈敏度 | 0.12μV / D |
| 荷重元激發電源 | 5 VDC ± 5%, 300mA（可接 8 個 350Ω 荷重元） |
| A/D 內部解析度 | 1,040,000 |
| 外部最大解析度 | 1/30,000 |
| A/D 轉換速度 | 10 次/秒 |

### 1.2 內建介面

| 介面 | 類型 |
|------|------|
| RS-232C | 內建 |
| RS-485 | 內建 |
| Current Loop | 內建 |
| DATA OUT | 選配 |

### 1.3 RS-232C / Current Loop 接腳定義

後面板接腳配置（DB-25 母座）：

| 接腳 | 名稱 | 方向 | 說明 |
|------|------|------|------|
| 2 PIN | TXD | LT-100 → PC | 傳送資料 |
| 3 PIN | RXD | PC → LT-100 | 接收資料（COMMAND 模式用） |
| 7 PIN | GND | — | 訊號地線 |
| 12 PIN | Current Loop | LT-100 → PC | 電流迴路傳送 |
| 13 PIN | Current Loop | PC → LT-100 | 電流迴路接收 |

> PC 端接線時，TXD 接 PC 的 RXD，RXD 接 PC 的 TXD，GND 對接。

### 1.4 DATA OUT 接腳定義（選配）

| 接腳 | 名稱 |
|------|------|
| 1 | +5V |
| 2 | CLK |
| 3 | DATA |

## 二、通訊參數設定（SF2）

進入方式：開機按住 `ESC` 不放 → 進入功能設定模式 → SF2

### 2.1 Baudrate（鮑率）

支援 7 種速率，使用按鍵切換：

| 選項 | 說明 |
|------|------|
| 1200 | |
| **2400** | **出廠預設值** |
| 4800 | |
| 9600 | |
| 19200 | |
| 38400 | |
| 57600 | |

### 2.2 Parity Check（同位檢查）

支援 3 種組合：

| 選項 | 格式說明 | 出廠預設 |
|------|----------|----------|
| **E.7.1** | Even Parity, 7 Data Bits, 1 Stop Bit | **✅ 預設** |
| N.8.1 | No Parity, 8 Data Bits, 1 Stop Bit | |
| O.7.1 | Odd Parity, 7 Data Bits, 1 Stop Bit | |

> 程式端的串列埠設定必須與 LT-100 的 SF2 設定完全一致，否則無法正確通訊。

| 選項 | 說明 |

## 四、Output Port 工作模式（SF4）

進入方式：SF4

此設定決定 LT-100 如何將資料傳送至電腦端，**是程式能否收到資料的關鍵設定**。

| 選項 | 說明 | 適用場景 |
|------|------|----------|
| **連續傳送** | 持續不斷傳送重量資料 | **✅ 出廠預設，ERPCore2 建議使用** |
| 歸零後穩定傳送一次 | 重量歸零後，穩定時傳送一次，回零點後再傳一次 | 單次計量 |
| 穩定後連續傳送 | 重量穩定後持續傳送，不穩定時停止 | 可用，但不穩定期間無即時資料 |
| 電腦端連結（COMMAND 模式） | 不主動傳送，等待 PC 發送命令後才回應 | 需程式主動發送指令 |

### ⚠️ 重要提醒

ERPCore2 採用**被動監聽模式**（訂閱 SerialPort.DataReceived 事件），不會主動發送命令。因此：

- SF4 設為「連續傳送」或「穩定後連續傳送」時，程式可正常接收
- SF4 設為「電腦端連結」時，**程式將完全無法接收到任何資料**

## 五、傳送格式設定（SF5）

進入方式：SF5

定義 RS-232C 傳送的資料格式結構。

### 5.1 四種傳送格式

| 格式 | 資料範例 | 說明 |
|------|----------|------|
| **Format 1** | `ST,GS,+03000.0kg` | **出廠預設**。狀態 + 模式 + 正負號數值單位連寫 |
| Format 2 | `ST,GS,+03000.0,kg` | 單位前有逗號，無小數點時補 `0` |
| Format 3 | `+03000.0kg` | 僅正負號 + 數值 + 單位，無狀態/模式前綴 |
| Format 4 | `ST,GS,+03000.0,kg` | 與 Format 2 類似，無小數點時補空白 |

### 5.2 Format 1 資料格式詳解（預設格式）

```
ST,GS,+03000.0kg CR LF
│  │  │ │       │  └─── 結尾符號：CR (0x0D) + LF (0x0A)
│  │  │ │       └────── 單位：kg / lb / g / t
│  │  │ └────────────── 重量值：7 位數（6 位數值 + 小數點），無小數點時補 "0"
│  │  └──────────────── 正負號："+" 正值 / "-" 負值
│  └─────────────────── 重量模式：GS = 總重 (Gross) / NT = 淨重 (Net)
└────────────────────── 穩定狀態：ST = 穩定 / US = 不穩定 / OL = 過載
```

### 5.3 穩定狀態代碼

| 代碼 | 含義 | 說明 |
|------|------|------|
| ST | Stable（穩定） | 重量已穩定，可信賴此數值 |
| US | Unstable（不穩定） | 重量仍在變動中 |
| OL | Overload（過載） | 超過最大秤量，數值不可用 |

### 5.4 重量模式代碼

| 代碼 | 含義 | 說明 |
|------|------|------|
| GS | Gross（總重） | 未扣除皮重的原始重量 |
| NT | Net（淨重） | 已扣除皮重的淨重量 |

### 5.5 單位代碼

| 代碼 | 含義 |
|------|------|
| kg | 公斤 |
| lb | 磅 |
| g | 公克 |
| t | 公噸 |

> 單位由 LT-100 的 CF5 設定決定，出廠預設為 kg。

## 六、資料格式設定（SF6）

進入方式：SF6

控制傳送資料中正負號與前導零的顯示方式。

| 選項 | 正負號 | 前導零 | 範例 | 對應機型格式 |
|------|--------|--------|------|-------------|
| **有 +- 有 0000** | ✅ | ✅ | `+03000.0` | **LT100 格式（出廠預設）** |
| 有 +- 無 0000 | ✅ | ❌ | `+3000.0` | 1705N 格式 |
| 無 +- 有 0000 | ❌ | ✅ | `03000.0` | 2001 格式 |
| 無 +- 無 0000 | ❌ | ❌ | `3000.0` | 1705 格式 |

### SF6 附加設定

#### 電腦端連結指令模式

| 選項 | 說明 |
|------|------|
| 關閉指令回傳 | **出廠預設** |
| 開啟指令回傳 | 啟用 PC COMMAND 模式指令回應 |

#### 資料輸出速度

預設值為 `01`，數值越大傳送速度越慢。

## 七、PC 命令模式（COMMAND 模式）

> 注意：需將 SF4 設為「電腦端連結」，且 SF6 中「指令回傳」設為開啟。
> ERPCore2 目前不使用此模式，但記錄於此供未來擴展參考。

PC 端透過 RS-232C 發送指令，LT-100 回應結果。所有指令以 `CR LF` (0x0D 0x0A) 結尾。

| 指令 | 發送格式 | 功能說明 |
|------|----------|----------|
| RD | `RD CR LF` | 讀取目前重量值（回傳格式同 SF5 設定） |
| READ | `READ CR LF` | 讀取目前重量值（同 RD） |
| ZO | `ZO CR LF` | 執行「歸零」動作（等同按 Zero 鍵） |
| TR | `TR CR LF` | 執行「扣重」動作（等同按 Tare 鍵） |
| GN | `GN CR LF` | 切換毛/淨重顯示（等同按 Gross/Net 鍵） |
| GS | `GS CR LF` | 切換至毛重 (Gross) 模式 |
| NT | `NT CR LF` | 切換至淨重 (Net) 模式 |
| PRN | `PRN CR LF` | 執行「列印」動作（等同按 Print 鍵） |

> 指令不區分大小寫（`RD` 與 `rd` 等效）。

## 八、數位濾波器設定（SF1）

進入方式：SF1

控制 A/D 轉換後的數位濾波強度，影響重量讀數的穩定性與反應速度。

| 段數 | 反應速度 | 穩定性 | 說明 |
|------|----------|--------|------|
| 關閉 | 最快 | 最低 | 無濾波 |
| 第 1 段 | 快 | 低 | |
| **第 4 段** | 中等 | 中等 | **出廠預設** |
| 第 10 段 | 最慢 | 最高 | |

> 段數越高，讀數越穩定但反應越慢。若秤台環境有震動干擾，可調高濾波段數。

## 九、其他影響傳輸的設定

### 9.1 DISPLAY 顯示亮度調整（SF7）

提供 4 段亮度調整（亮度一～亮度四），不影響 RS-232C 傳輸。

### 9.2 OP5L 時間差設定（SF8）

預設值為 3，數值越小傳送越快。影響 Output Port 的資料傳送間隔。

### 9.3 密碼設定（SF9）

預設值為 0（無密碼）。設定密碼後，進入功能設定模式需先輸入密碼。不影響 RS-232C 傳輸。

## 十一、常見問題排除

| 症狀 | 可能原因 | 解決方式 |
|------|----------|----------|
| 完全收不到資料 | SF4 設為「電腦端連結」 | 將 SF4 改為「連續傳送」 |
| 完全收不到資料 | COM Port 選錯 | 在裝置管理員確認正確的 COM Port |
| 完全收不到資料 | Baudrate / Parity 不匹配 | 確認程式端設定與 LT-100 SF2 一致 |
| 收到亂碼 / 解析失敗 | Baudrate 或 Parity 不匹配 | 確認雙方設定完全一致 |
| 解析失敗但有收到資料 | SF5/SF6 格式組合異常 | 檢查 LT-100 的 SF5、SF6 設定 |
| 讀數跳動不穩定 | 數位濾波器段數太低 | 調高 SF1 濾波段數 |
| 長時間無法等到穩定值 | 數位濾波器段數太高 / 環境干擾 | 降低 SF1 段數或排除干擾源 |
| 顯示 OL（過載） | 超過最大秤量 | 檢查秤台載重是否超過全秤量設定 |

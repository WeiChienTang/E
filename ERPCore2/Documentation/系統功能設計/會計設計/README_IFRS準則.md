# IFRS 準則參考文件

## 更新日期
2026-03-21

## 說明

本文件以 **ERP 系統設計視角**整理 IFRS 與台灣 TIFRS 的關鍵準則，供開發人員在設計帳務功能時參照。
每個準則章節均包含「台灣特殊規定」及「ERPCore2 現況與缺口」，便於快速確認系統是否合規。

> **名詞說明：**
> - **IFRS**：International Financial Reporting Standards（國際財務報導準則）
> - **IAS**：International Accounting Standards（國際會計準則，舊版準則，IFRS 前身，目前仍有效）
> - **TIFRS**：Financial Supervisory Commission (FSC) 金管會正式核准採用的繁體中文版 IFRS（部分條文與國際版有差異）
> - **ARDF**：財團法人中華民國會計研究發展基金會（負責翻譯並向金管會提交 TIFRS 版本）

## 一、台灣 TIFRS 採用現況

### 適用時程

| 適用對象 | 強制採用日期 |
|---------|------------|
| 上市、上櫃公司及金管會監管金融機構 | 2013 年 1 月 1 日 |
| 公開發行（興櫃）公司、信用合作社、信用卡公司 | 2015 年 1 月 1 日 |
| IFRS 9（金融工具）全面採用 | 2018 年 1 月 1 日 |
| IFRS 17（保險合約）—台灣延後 | **2026 年 1 月 1 日**（IASB 原為 2023） |
| IFRS 18（取代 IAS 1）—全球 | **2027 年 1 月 1 日**（預期金管會核可） |

### 台灣特殊差異（TIFRS vs. 全版 IFRS）

| 項目 | 全版 IFRS | 台灣 TIFRS | ERP 影響 |
|------|----------|-----------|---------|
| 不動產/廠房/設備重估（IAS 16） | 允許「重估模式」 | **取消**，只允許成本模式 | 固定資產不需支援向上重估 |
| 無形資產重估（IAS 38） | 活絡市場時可重估 | **取消** | 無形資產一律攤銷成本 |
| IFRS 17 保險合約 | 2023/01/01 | **延至 2026/01/01** | 保險業 ERP 有緩衝期 |
| 新準則生效日 | 依 IASB 公告 | 需待金管會核准公告，可能有時間差 | ERP 應追蹤金管會版本，不能直接套用 IASB 日期 |

### 非公開公司

非公開發行公司在台灣可選擇：
1. 金管會核准之 TIFRS（強烈建議，便於未來上市）
2. ARDF 另訂之「非公開發行公司適用準則」（簡化版）

**ERPCore2 定位**：中小型企業 ERP，採用 TIFRS 完整版為設計基準。

## 二、財務報表呈現（IAS 1）

### 完整財務報表組成

依 IAS 1，完整財務報表應包含：
1. **財務狀況表（資產負債表）** — 期末時點
2. **綜合損益表** — 期間損益 + 其他綜合損益（OCI）
3. **權益變動表** — 期間
4. **現金流量表**（IAS 7）— 期間
5. **附註** — 重要會計政策及說明
6. 若有追溯調整，須提供最早比較期間的期初財務狀況表

所有報表必須提供**比較期間**數字（通常為前一年度）。

### 財務狀況表最低呈現項目

**資產：**
- 不動產、廠房及設備（PP&E）
- 投資性不動產
- 無形資產（含商譽）
- 以權益法衡量之投資
- 生物資產
- 存貨
- 應收帳款及其他應收款
- 現金及約當現金
- 遞延所得稅資產
- 待出售非流動資產（IFRS 5）

**負債及權益：**
- 應付帳款及其他應付款
- 金融負債
- 準備（IAS 37）
- 遞延所得稅負債
- 流動所得稅負債
- 負債（IFRS 16 租賃負債）
- 股本及資本公積
- 保留盈餘
- 其他綜合損益累計數（OCI）
- 非控制權益（合併報表）

流動/非流動分類為強制要求（銀行業除外）。

### ERPCore2 現況

| 項目 | 狀態 | 說明 |
|------|------|------|
| 資產負債表（FN008）| ✅ 已實作 | 按 AccountType 分類顯示 |
| 損益表（FN007）| ✅ 已實作 | 收入/費用類科目彙總 |
| 現金流量表（FN014）| ❌ 未實作 | Phase 4-A 預計實作 |
| 權益變動表 | ❌ 未實作 | 未在路線圖中 |
| 比較期間數字 | ❌ 未實作 | 報表目前無跨年比較欄位 |
| 附註 | ❌ 未實作 | 超出目前 ERP 範圍 |

## 三、現金流量表（IAS 7）

### 三大類別

| 類別 | 說明 | 常見項目 |
|------|------|---------|
| **營業活動** | 主要營業產生的現金流量 | 向客戶收取款項、向供應商付款、支付員工薪資、繳納所得稅 |
| **投資活動** | 取得或處分長期資產 | 購置/出售 PP&E、購置/出售投資、借出/收回借款 |
| **籌資活動** | 改變業主與借款人資本結構 | 現金增資、舉借/償還借款、支付股利、償還租賃負債（IFRS 16） |

### 直接法 vs. 間接法

| 比較 | 直接法 | 間接法 |
|------|--------|--------|
| 起點 | 客戶收款、供應商付款等現金總額 | 本期稅後淨利 |
| IASB 建議 | 鼓勵（資訊更透明） | 允許 |
| 實務採用 | 少數公司 | 大多數公司 |
| ERPCore2 建議 | 間接法（較易從現有傳票資料推算） | — |

### 間接法調整項目

```
本期稅前淨利（FN007）
  加回：折舊與攤銷（非現金費用）
  加回：ECL 損失準備提列
  調整：應收帳款增減
  調整：存貨增減
  調整：應付帳款增減
  調整：預付/預收款增減
  減除：投資損益（移至投資活動）
  減除：利息費用（移至籌資活動）
─────────────────────
= 營業活動現金流量

投資活動：
  購置 PP&E                   (xxx)
  處分 PP&E                    xxx
─────────────────────
= 投資活動現金流量

籌資活動：
  現金增資                     xxx
  舉借長期借款                  xxx
  償還借款                    (xxx)
  支付租賃負債（IFRS 16）      (xxx)
  支付股利                    (xxx)
─────────────────────
= 籌資活動現金流量

本期現金及約當現金淨增減
+ 期初現金及約當現金
= 期末現金及約當現金（應等於 FN008 現金科目餘額）
```

### 科目分類映射（ERP 實作關鍵）

現金流量表的間接法需要知道「哪個科目屬於哪個類別的調整項目」。
建議在 `AccountItem` 上增加 `CashFlowCategory` 欄位：

| 類別 | 說明 | 代碼範例 |
|------|------|---------|
| `Operating_WorkingCapital` | 營業活動流動資金調整 | 應收帳款、存貨、應付帳款 |
| `Operating_NonCash` | 非現金費用（加回） | 折舊、攤銷、ECL 提列 |
| `Investing` | 投資活動 | PP&E、長期投資 |
| `Financing` | 籌資活動 | 長期借款、股本 |
| `Cash` | 現金及約當現金 | 1111~1121 |
| `Excluded` | 不含入現金流量（如損益轉入保留盈餘） | 結帳科目 |

### ERPCore2 現況

| 項目 | 狀態 | 說明 |
|------|------|------|
| 現金流量表（FN014）| ❌ 未實作 | Phase 4-A |
| `AccountItem.CashFlowCategory` | ❌ 未實作 | 實作 FN014 前需先建立此映射 |
| 期末現金餘額驗證 | ❌ 未實作 | 期末現金應等於 FN008 現金科目加總 |

## 四、存貨（IAS 2）

### 衡量原則：成本與淨變現價值孰低

```
帳面價值 = min(成本, NRV)
NRV（淨變現價值） = 估計售價 - 完工成本 - 銷售費用
```

### 允許的成本流動假設

| 方法 | IFRS 允許 | 說明 |
|------|----------|------|
| 先進先出（FIFO） | ✅ | 最常用；適合易腐品 |
| 加權平均成本（移動加權平均） | ✅ | ERPCore2 目前採用 |
| 個別認定法 | ✅ | 限不可互換的個別存貨（如藝術品、訂製品） |
| **後進先出（LIFO）** | ❌ **明確禁止** | IAS 2 不允許，TIFRS 同樣禁止 |

> **⚠️ 重要**：ERPCore2 採用移動加權平均（`ReduceStockAsync` 使用 `AverageCost`），符合 IAS 2。
> 系統**不可**提供 LIFO 選項。

### 跌價損失與迴轉

- 每期末評估是否有跌價（損壞、過時、售價下跌）
- 跌價損失計入損益（通常在銷貨成本內）
- **若跌價原因消失，必須迴轉**（不超過原始成本）

### ERPCore2 現況

| 項目 | 狀態 | 說明 |
|------|------|------|
| 移動加權平均 | ✅ 合規 | `ReduceStockAsync` 使用 `Product.AverageCost` |
| LIFO 選項 | ✅ 無此功能（合規） | — |
| 期末 NRV 評估 | ❌ 未實作 | 無存貨跌價準備功能 |
| 跌價損失傳票 | ❌ 未實作 | 需要手動輸入傳票 |

## 五、收入認列（IFRS 15）

### 五步驟模型

| 步驟 | 說明 |
|------|------|
| **Step 1** 識別合約 | 書面/口頭/默示合約，具商業實質，雙方均已核准 |
| **Step 2** 識別履約義務 | 每項「獨立」的商品/服務為一個履約義務 |
| **Step 3** 決定交易價格 | 預期可收取金額（扣除給客戶折讓、可變對價限制後） |
| **Step 4** 分攤交易價格 | 按各履約義務相對獨立售價分攤 |
| **Step 5** 認列收入 | 履約義務滿足時（某時點：控制權移轉；或一段時間：持續服務） |

### 預收款（合約負債）

客戶先付款但商品/服務未交付 → 認列為**合約負債（Contract Liability）**，不可直接認列收入。

### ERPCore2 現況

| 項目 | 狀態 | 說明 |
|------|------|------|
| 出貨時認列收入（某時點）| ✅ 合規 | `JournalizeSalesDeliveryAsync` 在出貨確認時建立傳票 |
| 預收款（合約負債科目）| 🟠 基礎支援 | 科目 2221（預收款項）存在，但無 IFRS 15 合約負債流程 |
| 多重履約義務分攤 | ❌ 未實作 | 超出目前 ERP 範圍，需單獨合約管理模組 |

## 六、租賃（IFRS 16）

### 承租人單一模型

IFRS 16（2019/01/01 生效）廢除融資/營業租賃區分，承租人一律認列：
- **使用權資產（ROU Asset）**
- **租賃負債（Lease Liability）**

例外免認列：
- **短期租賃**：租賃期 ≤ 12 個月
- **低價值資產**：資產原值 ≤ USD 5,000（約 NTD 160,000）

### 初始衡量

```
租賃負債 = 未來租賃付款額現值（折現率：隱含利率或增量借款利率）
使用權資產 = 租賃負債 + 預付租金 + 初始直接成本 + 復原義務 - 已收租賃獎勵
```

### ERPCore2 現況

| 項目 | 狀態 | 說明 |
|------|------|------|
| 租賃登記冊（Lease Register）| ❌ 未實作 | 無專屬 Entity |
| 使用權資產科目 | ❌ 未設定 | 種子資料中無 ROU 相關科目 |
| 攤銷排程 | ❌ 未實作 | — |
| 短期/低價值租賃費用化 | 🟠 | 可手動輸入傳票，但無系統判斷邏輯 |

> **建議**：租賃模組列入 Phase 4 或獨立規劃，中小企業若租賃資產不多可暫時手動輸入傳票處理。

## 七、金融工具（IFRS 9）—應收/應付帳款

### 金融資產分類

| 類別 | 衡量方式 | 應用場景 |
|------|---------|---------|
| **攤銷後成本（AC）** | 成本 + 有效利率利息 - 還款 ± ECL | 應收帳款、銀行存款（**最常用**） |
| FVOCI（含迴轉）| 公允價值，未實現損益入 OCI | 持有至收取及出售的債務工具 |
| FVOCI（不迴轉）| 公允價值，未實現損益入 OCI | 策略性股權投資 |
| **公允價值損益（FVTPL）** | 公允價值，損益入損益表 | 交易目的持有之有價證券 |

> **應收/應付帳款**：幾乎全部適用「攤銷後成本」，符合 SPPI 測試（只收取本金及利息）。

### IFRS 9 ECL 模型（應收帳款減損）

IFRS 9 採**預期信用損失（ECL）**模型，不再等到損失已發生才認列。

#### 簡化法（應收帳款強制適用）

對於無重大融資成分的應收帳款，自初始認列起即認列**整個存續期間的 ECL**。

#### 備抵矩陣法（實務工具）

依帳齡區間設定損失率：

| 帳齡區間 | 建議損失率（參考值） | 說明 |
|---------|------------------|------|
| 未逾期 | 0.3%～1% | 依歷史壞帳率調整 |
| 逾期 1～30 天 | 1%～3% | — |
| 逾期 31～60 天 | 3%～8% | — |
| 逾期 61～90 天 | 8%～20% | — |
| 逾期 91～180 天 | 20%～50% | — |
| 逾期 180 天以上 | 50%～100% | — |

> 損失率需依**歷史壞帳資料 + 前瞻性經濟調整**設定，非固定數字。

#### ECL 相關會計分錄

**每期末提列（備抵呆帳）：**
```
借：信用損失費用（Income Statement）  xxx
貸：備抵損失準備（Allowance for ECL）  xxx
```

**認定無法收回（沖銷）：**
```
借：備抵損失準備  xxx
貸：應收帳款      xxx
```

**後續實際收回：**
```
借：應收帳款      xxx
貸：備抵損失準備  xxx  ← 迴轉
```

### ERPCore2 現況

| 項目 | 狀態 | 說明 |
|------|------|------|
| AR/AP 帳齡分析（FN012/FN013）| ✅ 已實作 | 支援帳齡區間查詢 |
| ECL 損失率設定 | ❌ 未實作 | 無帳齡損失率參數設定 |
| ECL 備抵提列傳票 | ❌ 未實作 | 需手動輸入傳票 |
| 備抵損失準備科目 | ❌ 未設定 | 種子資料中可查找但尚未與 ECL 流程整合 |

> **建議**：ECL 計算與備抵提列可列為 Phase 4 延伸功能（FN012/FN013 的進一步深化）。

## 十、設計建議與優先順序

### 近期（Phase 4 之內）
1. **現金流量表 FN014**（IAS 7）— 需先在 `AccountItem` 增加 `CashFlowCategory` 欄位
2. **ECL 損失率設定頁面** — 在 FN012 帳齡報表旁增加備抵提列功能（IFRS 9 簡化法）

### 中期
3. **IFRS 18 準備**（2027 生效）— 損益表需重組為「營業/投資/籌資」三區塊；建議 2026 年開始規劃
4. **存貨 NRV 跌價準備**（IAS 2）— 期末評估與備抵傳票自動產生

### 長期 / 視業務需求
5. **租賃模組**（IFRS 16）— 使用權資產攤銷排程
6. **多幣別**（IAS 21）— 外匯重評與換算差
7. **固定資產模組**（IAS 16）— 含減損測試（IAS 36）

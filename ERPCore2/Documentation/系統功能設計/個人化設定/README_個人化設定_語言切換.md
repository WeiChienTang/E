# 個人化設定 — 語言切換設計

## 更新日期
2026-03-07

---

## 概述

Blazor Server 使用 SignalR 長連線，culture 在 HTTP 請求建立時即已確定，無法在連線存續期間動態切換。語言切換流程必須是：

**儲存語言偏好至 DB → 比對目前 UI 文化 → 寫入 culture cookie → `window.location.reload()`**

---

## 四層架構

| 層 | 說明 |
|----|------|
| **DB** `EmployeePreference.Language` | 持久化使用者偏好 |
| **Cookie** `.AspNetCore.Culture=c=zh-TW\|uic=zh-TW` | 登入後 / 語言變更後寫入 |
| **Middleware** `UseRequestLocalization` | 每次 HTTP 請求時設定 `CultureInfo.CurrentUICulture` |
| **Component** `IStringLocalizer<SharedResource>` | 元件渲染時取得本地化字串 |

---

## 支援語言清單

| UILanguage 枚舉 | Culture Code | 顯示名稱 | .resx 檔案 |
|----------------|-------------|---------|------------|
| `ZhTW = 1` | `zh-TW` | 繁體中文 | `SharedResource.resx`（預設） |
| `EnUS = 2` | `en-US` | English | `SharedResource.en-US.resx` |
| `JaJP = 3` | `ja-JP` | 日本語 | `SharedResource.ja-JP.resx` |
| `ZhCN = 4` | `zh-CN` | 简体中文 | `SharedResource.zh-CN.resx` |
| `FilPH = 5` | `fil` | Filipino | `SharedResource.fil.resx` |

## 核心檔案

| 檔案 | 說明 |
|------|------|
| `Resources/SharedResource.cs` | IStringLocalizer 定位用的空 marker class（namespace 必須為根命名空間 `ERPCore2`，不可為 `ERPCore2.Resources`） |
| `Resources/SharedResource*.resx` | 各語言翻譯字串（key 採英文點記法） |
| `Helpers/CultureHelper.cs` | `UILanguage` ↔ culture code 轉換 + `SupportedCultures` 陣列 |
| `wwwroot/js/culture-helper.js` | `setCultureAndReload(culture)` — 寫入 cookie 並執行 `window.location.reload()` |

## Resource Key 命名規則

| 前綴 | 用途 | 範例 |
|------|------|------|
| `Nav.*` | 導航選單項目與 NavMenu UI 文字 | `Nav.Employees`、`Nav.SignOut` |
| `Button.*` | 按鈕文字 | `Button.Save`、`Button.ConvertToReceiving` |
| `Label.*` | 一般標籤 | `Label.Actions`、`Label.Remarks` |
| `Modal.*` | Modal 標題 / 確認訊息 | `Modal.UnsavedChanges`、`Modal.ConfirmReject` |
| `Message.*` | 格式字串訊息（含 `{0}` 佔位符）| `Message.EditTitle`、`Message.UpdateSuccess` |
| `Entity.*` | 實體名稱 | `Entity.PurchaseOrder`、`Entity.Customer` |
| `Field.*` | 表單欄位標籤 | `Field.PurchaseOrderCode`、`Field.TaxRate` |
| `Column.*` | 表格欄位標題 | `Column.Quantity`、`Column.UnitPrice` |
| `Placeholder.*` | 輸入提示（含格式字串）| `Placeholder.PleaseInput`、`Placeholder.AutoGenerated` |

| `Option.*` | 下拉選項文字 | `Option.Sequential` |
| `Module.*` | 模組停用訊息 | `Module.NotAvailable`、`Module.DisabledMessage` |
| `Report.*` | 報表名稱與報表 UI 文字 | `Report.CustomerStatement`、`Report.GoTo` |
| `ReportCategory.*` | 報表集標題 | `ReportCategory.Customer` |
| `Page.*` | Index 頁面標題 / 副標題 | `Page.EmployeesTitle` |

## 重要設計規則

### 1. AddLocalization() 不可設 ResourcesPath

`SharedResource.cs` 與 `.resx` 同在 `Resources/` 資料夾，SDK 編譯後 embedded resource 名稱為 `ERPCore2.SharedResource`（以 class 的實際 namespace 命名）。若設 `ResourcesPath = "Resources"`，localizer 會尋找 `ERPCore2.Resources.SharedResource` → 找不到 → 回傳 key 名稱。必須使用 `builder.Services.AddLocalization()`（無參數）。

### 2. 語言切換與 CultureInfo.CurrentUICulture 比對

比對目前語言時，必須與 `CultureInfo.CurrentUICulture.Name`（目前 cookie 的文化）比對，而非與 DB 原始值比對。原因：若前次儲存已更新 DB 但 JS reload 失敗（cookie 未更新），下次開啟時若與 DB 值比對，DB 值等於目標值 → 永遠不觸發 reload。

### 3. IStringLocalizer 必須在 OnInitialized() 後使用

`L[]` 不可在欄位宣告（class field initializer）處使用，因 DI 尚未注入。`FormFieldDefinition` 的 `Label`、`Placeholder`，以及 `fieldSections` 字典的區段名稱，均須在 `OnInitialized()` 中初始化。若原本使用 `private readonly` 內聯初始化，需改為非 readonly 並移至 `OnInitialized()`。

### 4. `@using ApexCharts` 不可加入 `_Imports.razor`

（已記錄於圖表設計文件，此處不重複）

### 5. `[Parameter]` 預設值的本地化模式

`[Parameter]` 屬性無法在宣告時使用 `L[]`。解決方式：預設值改為 `""`，在使用處（`OnInitialized` 或 template）加 `L[]` fallback：`string.IsNullOrEmpty(ActionsHeader) ? L["Label.Actions"].ToString() : ActionsHeader`。

### 6. NavMenu 搜尋不隨語言變更

搜尋功能使用 `NavigationItem.Name`（繁體中文）與 `SearchKeywords`，結果名稱不隨使用者語言切換。可在 `SearchKeywords` 中補充多語言詞彙。

### 7. Entity.* 複用規則

欄位標籤與實體名稱完全相同時，直接複用 `Entity.*` key，不建立重複的 `Field.*`（例如 `Label = L["Entity.Customer"]` 而非新建 `Field.Customer`）。

### 8. Entity 雙 key 規則

部分實體的 EditModal 與 Index 頁面使用不同名稱時，分別建立兩個 key。例如 `Entity.PurchaseReceiving`（進貨，EditModal 用）與 `Entity.PurchaseReceivingDoc`（進貨單，Index 的 EntityName 用）。

### 9. Placeholder 格式字串模式

使用 3 個格式字串（`Placeholder.PleaseInput`、`Placeholder.PleaseSelect`、`Placeholder.PleaseInputOrSelect`）搭配 `Field.*` key 組合，減少重複 key。特定情境使用獨立 key（如 `Placeholder.AutoGenerated`、`Placeholder.SelectWarehouseFirst`）。

### 10. resx XML 實體字元

resx 是 XML 格式，值中含 `&` 字元（如 `Language & Region`）會導致 MSB3103 build error。所有值中的 `&` 必須寫成 `&amp;`。腳本生成 resx 後，必須抽查幾個 key 確認值是目標語言（而非英文 placeholder）。

## 新頁面翻譯流程

新增頁面或元件時依序：

1. 在所有 5 個語言的 `.resx` 加入新 `Entity.*` key（若實體尚未有對應 key）
2. 在目標元件注入 `@inject IStringLocalizer<SharedResource> L`
3. 替換 `GenericEditModalComponent` 參數：`EntityName`、`ModalTitle`（用 `Message.EditTitle` / `Message.CreateTitle`）、`SaveSuccessMessage`、`SaveFailureMessage`、`PrintButtonText`、`ReportPreviewTitle`
4. 若有自訂操作按鈕，先加入 `Button.*` key 再替換
5. `FormFieldDefinition` 的 `Label`、`Placeholder` 改用 `L["Field.*"]` / `L["Placeholder.*"]`（在 `OnInitialized` 中），優先複用 `Entity.*` key
6. Table 元件欄位標題改用 `L["Column.*"]`（Table 元件不自動帶入 L，需確認已注入）

---

## 新增語言 SOP

1. 在 `EmployeePreference.cs` 的 `UILanguage` 枚舉加入新值（int 從 6 遞增，不需 Migration）
2. 在 `CultureHelper.cs` 的 `ToCultureCode()` 和 `SupportedCultures` 陣列加入對應 culture code
3. 建立 `Resources/SharedResource.xx-XX.resx`，包含與預設 resx 相同的所有 key，值為目標語言（注意 `&` → `&amp;`）
4. 在 `LanguageRegionTab.razor` 的 `OnInitialized()` 語言選單 `Options` 清單加入新選項（`Value` 必須對應 UILanguage 枚舉 int 值）
